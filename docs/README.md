# Артефакты проекта интернет-магазина

Это рабочая папка с базовыми проектными артефактами. Они нужны, чтобы сначала закрепить правила магазина, а потом спокойно строить сайт, импорт остатков и будущую админку.

## Файлы

- `spec.md` - мини-ТЗ: роли клиентов, цены, скидки, доставка, заказы, импорт, ограничения первой версии.
- `data-model.md` - черновая модель данных: товары, клиенты, категории, акции, заказы, настройки, импорт.
- `import-workflow.md` - процесс импорта остатков из 1С 7.7 и правила сохранения ручных правок.

## Главные решения

- Первая версия работает локально на компьютере.
- Структура должна позволять позже переехать на VPS, CRM и админку.
- Источник остатков: `Остатки2.xls` из 1С 7.7.
- Курс по умолчанию: `89.00 сом` за `1 USD`.
- Базовая цена из 1С в USD считается оптовой.
- Розничная цена = оптовая цена в сомах + `30%`, затем красивое округление.
- Скидка зарегистрированного клиента считается от красивой розничной цены и округляется вниз до целого сома.
- Оптовики получают доступ только после подтверждения администратора.
- Заказ не оплачивается онлайн, а отправляется менеджеру в WhatsApp и email.
- Остатки клиенту не показываются.
- Резервирование товара не делается.
- Бесплатная доставка от `10000 сом`; ниже доставка согласовывается с менеджером.
- Товар может быть в нескольких категориях.
- Категории должны быть красивыми клиентскими, не обязательно как в 1С.
- Картинки пока заменяются красивыми заглушками; поиск реальных картинок только после отдельного согласования.

## Важное правило импорта

Новый файл из 1С должен обновлять только учетные данные:

- остаток
- базовую USD-цену
- сумму
- группу из 1С
- дату импорта

Он не должен затирать витринные улучшения:

- красивое название
- описание
- характеристики
- категории
- теги
- картинки
- скрытие товара
- акции
- ручные настройки
