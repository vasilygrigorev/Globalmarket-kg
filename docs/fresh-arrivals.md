# Свежие поступления из 1С

## Правило витрины

«Свежие товары» — только позиции, чей стабильный персональный код 1С впервые
появился в последней выгрузке остатков. Изменение названия, цены или количества
не делает существующий товар новым.

- `firstSeenAt` — дата первого появления стабильного кода 1С;
- `latestStockDate` — дата последней импортированной выгрузки;
- товар попадает в блок, только если `firstSeenAt == latestStockDate`;
- старые записи без достоверной даты первого появления остаются без даты;
- если новых кодов в выгрузке нет, блок скрывается;
- `restockedAt` хранится отдельно и не определяет новизну товара.

## Идентичность товара

Основной идентификатор — персональный код позиции из 1С (`source_code`). Название
можно исправлять и дополнять без создания нового товара. Импорт сопоставляет строку
по стабильному коду и сохраняет прежний `first_seen_at`.

SQLite-таблица `source_products` получила колонку `first_seen_at`. При первом
импорте нового кода она получает дату отчёта 1С. При повторных импортах дата не
изменяется. Для существующей старой базы миграция добавляет колонку, но не ставит
сегодняшнюю дату всем старым товарам.

## Порядок показа

Новые позиции группируются по типу и выдаются круговым проходом. Приоритет начала:

1. порошок;
2. шампунь;
3. дезодорант или антиперспирант;
4. зубная щётка;
5. остальные типы товара.

После одной позиции из каждого доступного типа начинается следующий круг. Поэтому
варианты одной линейки не занимают всю карусель, а клиент сразу видит ассортимент
из разных разделов. Внутри каждого типа выше ставятся товары с фотографией, затем
по рейтингу и названию.

## Пополнение остатка

`restockedAt` продолжает обновляться отдельно, когда количество товара выросло или
товар вернулся с нулевого остатка. Поле можно использовать для внутренней аналитики
и сортировки пополнений, но не для публичного блока «Свежие товары».

## Поток данных

1. `scripts/import_stock.py` читает выгрузку 1С и сохраняет `first_seen_at`.
2. `generate_outputs()` пишет `firstSeenAt` и `latestStockDate` в
   `data/catalog.json`.
3. `scripts/build_public_catalog.py` переносит поля в публичный каталог.
4. `app.js` выбирает только новые активные позиции и чередует их по типам.

Правило начнёт наполнять блок после следующего реального импорта 1С. Текущим старым
товарам дата первого появления намеренно не назначается задним числом.
